# Repository Architecture Summary

## Overview

`unscramble` is a Next.js App Router project for Unscramble Word Now. The app helps users find valid words from scrambled letters, browse words by length, starting letters, ending patterns, and wildcard patterns, and expose SEO-friendly dynamic pages for those searches.

The project is built with Next.js 16, React 19, TypeScript, Tailwind CSS 4, and precompiled JSON word indexes.

## Top-Level Structure

```text
app/                  Next.js routes, layouts, API routes, sitemap, robots
components/           Shared UI components used across pages
data/                 Raw, source, compiled, and analytics word data
docs/                 Project documentation and planning notes
lib/                  Search engine, word lookup, and SEO helper code
public/               Static public assets
scripts/              Dictionary and SEO index build scripts
```

## Application Routes

The app uses the App Router under `app/`.

```text
app/page.tsx
```

Interactive homepage. Users enter scrambled letters, optional filters, and receive ranked matching words from the API route.

```text
app/api/unscramble/route.ts
```

API endpoint used by the homepage. It accepts `letters`, `length`, `startsWith`, `endsWith`, and `contains` query parameters, then returns matching words, the best word, and definitions.

```text
app/unscramble/[letters]/page.tsx
```

SEO-friendly dynamic page for a specific scrambled-letter query.

```text
app/word-finder/page.tsx
```

Server-rendered word finder form for length, prefix, suffix, and contains filters.

```text
app/words-from-letters/page.tsx
app/words-from-letters/[letters]/page.tsx
```

Release 2 pages for finding every valid word that can be made from a supplied set of letters. Results are grouped by word length and the dynamic route provides SEO metadata and related internal links.

```text
app/word-length/[length]/page.tsx
app/words-starting-with/[letter]/page.tsx
app/words-ending-with/[suffix]/page.tsx
app/words-with-pattern/[pattern]/page.tsx
```

Dynamic SEO route families for browsable word lists.

```text
app/sitemap.ts
app/robots.ts
```

Metadata routes for search engine discovery.

## Data Flow

The project is optimized around precomputed indexes rather than expensive runtime scans.

```text
data/source/dictionary.json
        |
        v
scripts/buildIndex.ts
        |
        v
data/source/words.txt
data/compiled/signatureMap.json
data/compiled/startsWithMap.json
data/compiled/endsWithMap.json
data/compiled/lengthMap.json
data/compiled/frequencyMap.json
data/compiled/definitions.json
```

Additional SEO pattern data is generated by:

```text
scripts/buildSEO.ts
        |
        v
data/compiled/patternIndex.json
```

At runtime, pages and API routes import the compiled JSON files directly through `lib/engine`.

## Search Engine Modules

```text
lib/engine/wordStore.ts
```

Primary lookup layer. It imports compiled maps, normalizes inputs, ranks words by frequency and length, applies filters, and exposes helpers for:

- unscrambling letters
- finding words by length
- finding words by prefix
- finding words by suffix
- retrieving definitions and frequency scores

```text
lib/engine/search.ts
```

Pattern lookup helper for wildcard pattern pages.

```text
lib/engine/autocomplete.ts
lib/engine/fuzzy.ts
```

Autocomplete support using the starts-with index plus lightweight fuzzy ranking.

```text
lib/engine/trending.ts
```

Static homepage trending word suggestions.

```text
lib/engine/wordsFromLetters.ts
```

Release 2 subset-letter engine. It checks dictionary words against available letter counts, reuses existing length and frequency data, memoizes base searches, supports shared filters, and groups results by word length.

```text
lib/engine/unscrambleEngine.ts
lib/engine/wordFilters.ts
```

Older or narrower lookup helpers that overlap with `wordStore.ts`.

## Shared UI Components

```text
components/SiteHeader.tsx
```

Global header included by `app/layout.tsx`.

```text
components/RelatedLinks.tsx
```

Generates internal links between related SEO pages.

Other UI components support word result display, input tiles, search boxes, and best-word presentation.

## SEO Architecture

SEO is handled through a combination of:

- route-level `metadata` and `generateMetadata`
- dynamic route families for high-intent word searches
- internal links via `RelatedLinks`
- sitemap generation in `app/sitemap.ts`
- robots configuration in `app/robots.ts`
- content helpers under `lib/seo`

The production domain is configured as:

```text
https://unscramblewordnow.com
```

## Build And Validation

Package scripts:

```text
npm run dev      Start local development server
npm run build    Build production app
npm run start    Start production server
npm run lint     Run ESLint
```

Before major changes, run:

```text
npm run build
```

The repository instructions also note that this Next.js version may have breaking changes. Before changing Next-specific APIs or conventions, check the relevant guide under:

```text
node_modules/next/dist/docs/
```

## Current Architectural Notes

- The app favors precomputed JSON indexes for fast lookups.
- `wordStore.ts` is the main shared engine boundary.
- Some helper modules duplicate functionality now centralized in `wordStore.ts`.
- Several docs and scripts contain mojibake text where symbols such as arrows or checkmarks appear incorrectly encoded.
- The roadmap currently prioritizes production readiness, SEO validation, route testing, internal linking, Search Console, and sitemap submission.
